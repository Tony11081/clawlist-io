Best Practices

01

Least Privilege Access

Only grant the minimum permissions required for each API key to function. Never use root keys for routine tasks.

02

Environment Isolation

Use separate API keys for development, staging, and production environments to prevent cross-contamination.

03

Secure Storage

Never hardcode API keys. Use environment variables or secret management tools like HashiCorp Vault.

04

Payload Encryption

Always use TLS 1.3 for data in transit and consider client-side encryption for sensitive payloads.

API Key Safety

check_circle The "Do's"
  • • Store keys in .env files ignored by Git
  • • Regularly monitor usage logs
  • • Set hard usage quotas
cancel The "Don'ts"
  • • Embed keys in frontend JavaScript
  • • Send keys via Slack or Email
  • • Use the same key for all apps
